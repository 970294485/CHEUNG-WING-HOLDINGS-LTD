# 開發階段拆解 (Development Phase)

> 本文檔將綜合企業管理系統 (ERP/CRM) 開發劃分為 6 個階段（Phase）。每個階段細分為可獨立完成的子任務，包含描述、關鍵實現和驗收標準。
>
> **讀者**：開發者、AI Agent。

---

## 開發前必讀

### 前置文檔

| 文檔 | 路徑 | 用途 |
|------|------|------|
| **業務邏輯** | `docs/business_logic.md` | 系統總覽、核心模塊業務邏輯、工作流、權限與審批 |
| **技術規格** | `docs/tech.md` | 技術棧、數據庫 Schema、API 接口、部署架構 |

**使用方式**：開發時 @ 本文件並說明要做的 Phase/任務編號。AI Agent 必須先讀取 `docs/business_logic.md` 和 `docs/tech.md` 再開始實現。

### 核心設計決策

| 決策 | 原因 | 實現 |
|------|------|------|
| **業財一體化** | 避免數據孤島，減少重複錄入 | 業務動作（出入庫、合同）通過事務自動生成財務憑證 |
| **動態產品屬性** | 產品種類繁多，規格不一 | 數據庫採用 JSONB 字段存儲自由增加、多選的屬性 |
| **統一文件存儲** | 集中管理企業數字資產 | S3/OSS 統一存儲，業務附件與網盤底層打通 |
| **RBAC 權限模型** | 企業級數據安全要求 | 中間件攔截 + API 細粒度校驗 |

## Phase 1：基礎架構與資料輸入模塊 [P0]

> 目標：搭建項目骨架，完成用戶、權限及基礎資料（1.7）的 CRUD，為後續業務模塊提供基石。

### 任務 1.1：項目骨架與數據庫初始化
**描述**：初始化 Next.js 項目，配置 Tailwind CSS、Shadcn UI，搭建 PostgreSQL 數據庫及 ORM。
**驗收標準**：
- [x] 項目可正常啟動，頁面路由正常。
- [x] 數據庫連接成功，ORM Schema 可遷移。

### 任務 1.2：身份認證與權限管理 (RBAC)
**描述**：實現用戶登錄、公司權限設定、級別權限設定 (1.7)。
**驗收標準**：
- [x] 支持用戶登錄/登出。
- [x] 可創建角色並分配權限（菜單級、按鈕級）。
- [x] 中間件能正確攔截無權限訪問。

### 任務 1.3：基礎資料錄入 (1.7)
**描述**：實現基本公司資料、公司基本文件單號規則、會計入賬項目與類別輸入的界面與 API。
**驗收標準**：
- [x] 公司信息可配置。
- [x] 單號生成規則可自定義（如 INV-YYYYMM-001）。
- [x] 會計科目/類別可增刪改查。

## Phase 2：產品與庫存模塊 [P0]

> 目標：建立實物資產檔案，實現庫存的動態流轉。

### 任務 2.1：產品管理 (1.8)
**描述**：實現產品檔案的錄入，包括基本信息、多媒體上傳、配件設定、動態屬性及價格。
**驗收標準**：
- [x] 支持 SKU、Barcode、規格等基礎字段。
- [x] 支持自由增加產品屬性並多選（JSONB 實現）。
- [x] 支持圖片/Video 附件上傳。
- [x] 支持產品目錄導出為 Excel/PDF。

### 任務 2.2：庫存管理 (1.9)
**描述**：實現出入庫功能、詳細庫存臺賬及成本查看。
**驗收標準**：
- [x] 可手動或通過單據觸發入庫、出庫操作。
- [x] 庫存明細實時更新，防超賣（併發控制）。
- [x] 支持查看庫存移動平均成本。

### 任務 2.3：期初倉存數據錄入 (1.7)
**描述**：提供界面批量導入或手動錄入期初庫存數據。
**驗收標準**：
- [x] 期初數據錄入後正確反映在庫存臺賬中。

## Phase 3：客戶與銷售模塊 [P1]

> 目標：實現“線索到現金 (Lead to Cash)”的完整業務閉環，涵蓋從獲客、報價、合同簽訂到開具預收發票的全過程。

### 任務 3.1：客戶管理 (1.6)
**描述**：實現客戶檔案管理、分組標籤、跟進記錄及郵件營銷集成。
**驗收標準**：
- [x] 客戶檔案 (CRUD) 及分組/標籤功能正常。
- [x] 可添加跟進記錄，並以時間軸 (Timeline) 視圖展示客戶生命週期。
- [x] 記錄並統計客戶來源（如展會、官網等）。
- [x] 集成 Email 發送推廣訊息功能。

### 任務 3.2：銷售單據流轉 (1.3)
**描述**：實現見積(報價)管理 -> 合同管理 -> 預收發票的自動化業務流轉。
**驗收標準**：
- [x] 報價單：可從產品庫選擇產品（帶出SKU/價格），自動計算總價。
- [x] 合同：報價單可一鍵轉為銷售合同，避免重複錄入。
- [x] 發票：基於銷售合同可直接生成預收發票。
- [x] 單據狀態聯動正常，支持上下游單據的穿透查看。

### 任務 3.3：銷售協同與導出 (1.3, 1.4)
**描述**：打通庫存與財務數據，實現防超賣、預收款核銷及單據標準化導出。
**驗收標準**：
- [x] 實時庫存對接：開單（報價/合同）時可查看實時可用庫存，庫存不足時提示。
- [x] 財務預收款對接：開具合同/發票時，可關聯並核銷客戶的預收款。
- [x] 佣金計算：銷售回款後，根據預設規則自動計算銷售佣金。
- [x] 單據導出：報價單、發票、送貨單提供統一模板，可一鍵導出為 PDF/Excel。

## Phase 4：財務與會計模塊 [P1]

**驗收標準**：

- [x] 預算管理功能正常，可設置收支預算。

### 任務 4.2：會計賬務管理 (1.2)
**描述**：實現總賬管理、應收(AR)與應付(AP)賬的管理。
**驗收標準**：

- [x] 支持手動錄入會計憑證。
- [x] 總賬明細清晰可查。

```
Phase 1: 基礎架構與資料輸入 [P0]
 ├── 1.1 項目骨架與數據庫
 ├── 1.2 身份認證與權限 (RBAC)
 └── 1.3 基礎資料錄入

Phase 2: 產品與庫存模塊 [P0] (依賴 Phase 1)
 ├── 2.1 產品管理
 ├── 2.2 庫存管理
 └── 2.3 期初倉存數據錄入

Phase 3: 客戶與銷售模塊 [P1] (依賴 Phase 1, 2)
 ├── 3.1 客戶管理
 ├── 3.2 銷售單據流轉
 └── 3.3 銷售協同與導出

Phase 4: 財務與會計模塊 [P1] (依賴 Phase 1, 2, 3)
 ├── 4.1 財務收支與審批
 └── 4.2 會計賬務管理

Phase 5: 文件管理與導出模塊 [P2] (依賴 Phase 1)
 ├── 5.1 個人網盤與公共數據庫
 └── 5.2 全局文件導出中心

Phase 6: 報表與數據分析 [P2] (依賴 Phase 1~4)
 ├── 6.1 財務與會計報表
 └── 6.2 業務分析圖表
```
