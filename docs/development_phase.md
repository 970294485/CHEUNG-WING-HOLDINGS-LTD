# 开发阶段拆解 (Development Phase)

> 本文档将综合企业管理系统 (ERP/CRM) 开发划分为 6 个阶段（Phase）。每个阶段细分为可独立完成的子任务，包含描述、关键实现和验收标准。
>
> **读者**：开发者、AI Agent。

---

## 开发前必读

### 前置文档

| 文档 | 路径 | 用途 |
|------|------|------|
| **业务逻辑** | `docs/business_logic.md` | 系统总览、核心模块业务逻辑、工作流、权限与审批 |
| **技术规格** | `docs/tech.md` | 技术栈、数据库 Schema、API 接口、部署架构 |

**使用方式**：开发时 @ 本文件并说明要做的 Phase/任务编号。AI Agent 必须先读取 `docs/business_logic.md` 和 `docs/tech.md` 再开始实现。

### 核心设计决策

| 决策 | 原因 | 实现 |
|------|------|------|
| **业财一体化** | 避免数据孤岛，减少重复录入 | 业务动作（出入库、合同）通过事务自动生成财务凭证 |
| **动态产品属性** | 产品种类繁多，规格不一 | 数据库采用 JSONB 字段存储自由增加、多选的属性 |
| **统一文件存储** | 集中管理企业数字资产 | S3/OSS 统一存储，业务附件与网盘底层打通 |
| **RBAC 权限模型** | 企业级数据安全要求 | 中间件拦截 + API 细粒度校验 |

## Phase 1：基础架构与资料输入模块 [P0]

> 目标：搭建项目骨架，完成用户、权限及基础资料（1.7）的 CRUD，为后续业务模块提供基石。

### 任务 1.1：项目骨架与数据库初始化
**描述**：初始化 Next.js 项目，配置 Tailwind CSS、Shadcn UI，搭建 PostgreSQL 数据库及 ORM。
**验收标准**：
- [x] 项目可正常启动，页面路由正常。
- [x] 数据库连接成功，ORM Schema 可迁移。

### 任务 1.2：身份认证与权限管理 (RBAC)
**描述**：实现用户登录、公司权限设定、级别权限设定 (1.7)。
**验收标准**：
- [x] 支持用户登录/登出。
- [x] 可创建角色并分配权限（菜单级、按钮级）。
- [x] 中间件能正确拦截无权限访问。

### 任务 1.3：基础资料录入 (1.7)
**描述**：实现基本公司资料、公司基本文件单号规则、会计入账项目与类别输入的界面与 API。
**验收标准**：
- [x] 公司信息可配置。
- [x] 单号生成规则可自定义（如 INV-YYYYMM-001）。
- [x] 会计科目/类别可增删改查。

## Phase 2：产品与库存模块 [P0]

> 目标：建立实物资产档案，实现库存的动态流转。

### 任务 2.1：产品管理 (1.8)
**描述**：实现产品档案的录入，包括基本信息、多媒体上传、配件设定、动态属性及价格。
**验收标准**：
- [x] 支持 SKU、Barcode、规格等基础字段。
- [x] 支持自由增加产品属性并多选（JSONB 实现）。
- [x] 支持图片/Video 附件上传。
- [x] 支持产品目录导出为 Excel/PDF。

### 任务 2.2：库存管理 (1.9)
**描述**：实现出入库功能、详细库存台账及成本查看。
**验收标准**：
- [x] 可手动或通过单据触发入库、出库操作。
- [x] 库存明细实时更新，防超卖（并发控制）。
- [x] 支持查看库存移动平均成本。

### 任务 2.3：期初仓存数据录入 (1.7)
**描述**：提供界面批量导入或手动录入期初库存数据。
**验收标准**：
- [x] 期初数据录入后正确反映在库存台账中。

## Phase 3：客户与销售模块 [P1]

> 目标：实现“线索到现金 (Lead to Cash)”的完整业务闭环，涵盖从获客、报价、合同签订到开具预收发票的全过程。

### 任务 3.1：客户管理 (1.6)
**描述**：实现客户档案管理、分组标签、跟进记录及邮件营销集成。
**验收标准**：
- [x] 客户档案 (CRUD) 及分组/标签功能正常。
- [x] 可添加跟进记录，并以时间轴 (Timeline) 视图展示客户生命周期。
- [x] 记录并统计客户来源（如展会、官网等）。
- [x] 集成 Email 发送推广讯息功能。

### 任务 3.2：销售单据流转 (1.3)
**描述**：实现見積(报价)管理 -> 合同管理 -> 预收发票的自动化业务流转。
**验收标准**：
- [x] 报价单：可从产品库选择产品（带出SKU/价格），自动计算总价。
- [x] 合同：报价单可一键转为销售合同，避免重复录入。
- [x] 发票：基于销售合同可直接生成预收发票。
- [x] 单据状态联动正常，支持上下游单据的穿透查看。

### 任务 3.3：销售协同与导出 (1.3, 1.4)
**描述**：打通库存与财务数据，实现防超卖、预收款核销及单据标准化导出。
**验收标准**：
- [x] 实时库存对接：开单（报价/合同）时可查看实时可用库存，库存不足时提示。
- [x] 财务预收款对接：开具合同/发票时，可关联并核销客户的预收款。
- [x] 佣金计算：销售回款后，根据预设规则自动计算销售佣金。
- [x] 单据导出：报价单、发票、送货单提供统一模板，可一键导出为 PDF/Excel。

## Phase 4：财务与会计模块 [P1]

> 目标：管理资金流，实现账务核算与审批控制。

### 任务 4.1：财务收支与审批 (1.1)
**描述**：管理请款单、预收款项，实现合同/发票与预收款的对接；设置审批流。
**验收标准**：
- [x] 员工可提交请款单，按规则触发审批流。
- [x] 财务可录入预收款，并在开票时进行核销对接。
- [x] 预算管理功能正常，可设置收支预算。

### 任务 4.2：会计账务管理 (1.2)
**描述**：实现总账管理、应收(AR)与应付(AP)账的管理。
**验收标准**：
- [x] 销售/采购动作自动生成 AR/AP 记录。
- [x] 支持手动录入会计凭证。
- [x] 总账明细清晰可查。

---

## Phase 5：文件管理与导出模块 [P2]

> 目标：提供企业级网盘及全局单据导出能力。

### 任务 5.1：个人网盘与公共数据库 (1.5)
**描述**：实现文件分类、上传下载、个人网盘及内部文件分享。
**验收标准**：
- [x] 员工拥有个人存储空间（上传/下载/分类）。
- [x] 支持将个人文件分享到公共数据库给内部员工。
- [x] 权限隔离：只能看自己和被分享的文件。

### 任务 5.2：全局文件导出中心 (1.4)
**描述**：统一管理发票、报价单、送货单、付款请求的导出功能。
**验收标准**：
- [x] 提供统一的导出服务（PDF/Excel）。
- [x] 导出的文件格式规范、样式美观。

---

## Phase 6：报表与数据分析 [P2]

> 目标：为管理层提供决策支持的数据看板。

### 任务 6.1：财务与会计报表 (1.1, 1.2)
**描述**：生成月度预算统计、利润和损失表 (P&L)、企业资产负债表。
**验收标准**：
- [x] P&L 数据计算准确（收入-成本-费用）。
- [x] 资产负债表借贷平衡。
- [x] 预算执行报告可视化展示。

### 任务 6.2：业务分析图表 (1.3, 1.6)
**描述**：生成客户分析图表、销售分析报告。
**验收标准**：
- [x] 客户来源、转化率图表展示。
- [x] 销售业绩趋势图表展示。

```
 └── 3.3 销售协同与导出

Phase 4: 财务与会计模块 [P1] (依赖 Phase 1, 2, 3)
 ├── 4.1 财务收支与审批

Phase 5: 文件管理与导出模块 [P2] (依赖 Phase 1)

Phase 6: 报表与数据分析 [P2] (依赖 Phase 1~4)

```
