# 業務邏輯文檔 (Business Logic)

> 本文檔是「綜合企業管理系統 (ERP/CRM)」的核心業務參考，涵蓋財務、會計、銷售、客戶、產品、庫存、文件管理及系統基礎設置等全鏈路業務邏輯。
>
> **核心理念**：業財一體化、數據互通、權限嚴控、流程自動化。
>
> **讀者**：產品、開發、運營及使用系統的企業員工。
>
> **相關文檔**：`docs/tech.md`（技術規格）、`docs/development_phase.md`（開發階段）

## 術語表

| 術語 | 說明 |
|------|------|
| **見積 (Quotation)** | 報價單，向客戶提供的產品/服務價格明細 |
| **AR (Accounts Receivable)** | 應收賬款，客戶欠企業的貨款 |
| **AP (Accounts Payable)** | 應付賬款，企業欠供應商的貨款 |
| **P&L (Profit & Loss)** | 利潤和損失表（損益表），反映企業一定期間的經營成果 |
| **SKU** | 庫存進出計量的基本單元 |
| **預收款 (Advance Payment)** | 在交付產品或服務前，提前向客戶收取的款項 |

## 1. 系統總覽

### 1.1 核心設計原則

| 原則 | 含義 |
|------|------|
| **業財一體化** | 銷售、採購、庫存的業務動作自動觸發財務記賬，減少人工干預 |
| **權限隔離** | 嚴格的級別權限設定，銷售看客戶，庫管看庫存，財務看賬目 |
| **靈活可配** | 產品屬性、會計類別、審批流等支持自定義，適應不同業務場景 |
| **數據資產化** | 完善的文件分類與網盤功能，沉澱企業知識與業務單據 |

### 1.2 系統模塊定位

| 模塊 | 說明 |
|------|------|
| **1.7 資料輸入 (PT-Data Entry)** | 系統的基石，負責公司基本資料、權限、用戶、產品主數據、期初數據的錄入 |
| **1.6 客戶管理 (Customer)** | 獲客與維繫，支持分組、跟進、郵件推廣及分析 |
| **1.8 產品管理 (Product)** | 定義實物資產，支持多屬性、多媒體、配件及價格體系 |
| **1.3 銷售功能 (Sales)** | 交易閉環，從報價(見積) -> 合同 -> 預收發票 -> 佣金計算 |
| **1.9 庫存管理 (Inventory)** | 實物管理，出入庫操作、庫存明細及成本核算 |
| **1.1 財務功能 (Financial)** | 資金流管理，請款、預收款核銷、預算控制及審批流 |
| **1.2 會計功能 (Accounting)** | 賬務核算，應收應付、總賬、P&L、資產負債表 |
| **1.4 / 1.5 文件與導出 (Document/File)** | 單據 PDF/Excel 導出、個人網盤與公共文件數據庫 |

## 2. 完整用戶工作流

### 2.1 核心業務閉環（線索到現金 Lead to Cash）

```
1. 客戶建檔 → 錄入客戶信息、分組、分配跟進人 (1.6)
2. 銷售報價 → 選擇產品(1.8)，生成並導出報價單 PDF (1.3, 1.4)
3. 簽訂合同 → 報價轉合同，確認交易細節 (1.3)
4. 預收與發票 → 生成預收發票，財務核銷預收款 (1.1, 1.3)
5. 備貨與採購 → 庫存對接，庫存不足則觸發採購單 (1.3, 1.7, 1.9)
6. 發貨出庫 → 生成送貨單(1.4)，扣減庫存(1.9)
7. 財務結算 → 生成應收賬款(AR)，確認收入，計算銷售佣金 (1.2, 1.3)
8. 賬務報表 → 數據自動流入總賬，體現在 P&L 和資產負債表中 (1.2)
```

### 2.2 採購到付款閉環（Procure to Pay）

```
1. 採購需求 → 銷售缺貨或日常備貨觸發採購單 (1.7)
2. 採購入庫 → 庫管核對實物，操作入庫，增加庫存並核算成本 (1.9)
3. 財務請款 → 業務發起請款單，進入審批流 (1.1)
4. 審批與付款 → 財務/管理層審批通過，執行付款，核銷應付賬款(AP) (1.1, 1.2)
```

## 3. 核心模塊業務邏輯詳解

### 3.1 財務與會計模塊 (Financial & Accounting)

**財務管理 (1.1)** 側重於資金的計劃與控制：
*   **請款與審批**：所有資金流出必須通過請款單發起，根據金額/類別觸發預設的審批權限設置。
*   **預收款對接**：客戶打款後，財務記錄預收款；銷售開具合同/發票時，系統支持將預收款與具體合同/發票進行核銷對接。
*   **預算管理**：設定收入與支出預算，系統自動按月度統計實際發生額，生成預算執行對比報告。

**會計功能 (1.2)** 側重於賬務的準確記錄：
*   **入賬與類別**：基礎數據中設定的會計入賬項目與類別，用於規範所有收支記錄。
*   **往來賬 (AR/AP)**：銷售出庫自動增加應收；採購入庫自動增加應付。
*   **核心報表**：系統根據總賬記錄，實時生成利潤和損失表 (P&L) 及企業資產負債表。

### 3.2 銷售與客戶模塊 (Sales & Customer)

*   **客戶生命週期 (1.6)**：支持客戶分組管理、跟進情況記錄、來源追蹤。集成 Email 發送推廣訊息功能。提供客戶分析圖表。
*   **交易管理 (1.3)**：
    *   見積 (Quotation) -> 合同 (Contract) -> 預收發票 (Proforma Invoice) 的單據流轉。
    *   **庫存對接**：銷售開單時實時查看庫存，確認是否可售。
    *   **財務對接**：銷售回款直接關聯財務模塊，並根據規則自動計算銷售佣金。

### 3.3 產品與庫存模塊 (Product & Inventory)

*   **產品主數據 (1.8)**：
    *   基礎字段：名稱、SKU、Barcode、型號、規格、類型/子類型、重量、尺寸、包裝尺寸、顏色。
    *   多媒體：支持圖片、Video、附件上傳。
    *   **高級特性**：支持產品配件設定；**產品屬性支持自由增加並可以多選**（如自定義材質、適用場景等）。
    *   價格管理與導出（Excel/PDF）。
*   **庫存管理 (1.9)**：
    *   管理出入庫動作，提供詳細庫存臺賬。
    *   **成本查看**：結合採購入庫價格，計算並展示庫存成本（如移動加權平均）。

### 3.4 文件與系統基礎 (Document, File & Data Entry)

*   **文件導出 (1.4)**：標準化輸出發票、報價單、送貨單、付款請求等（支持 Excel/PDF）。
*   **文件管理 1.0 (1.5)**：
    *   **個人網盤**：員工專屬，支持上傳、下載、檔案分類和分享。
    *   **公共數據庫**：企業知識庫，支持把檔案分享給內部員工。
*   **資料輸入 (1.7)**：
    *   系統初始化入口：公司基本資料、單號規則、會計類目。
    *   權限控制：公司權限設定、級別權限設定、用戶資料輸入。
    *   歷史/期初數據：報價單、合同、採購單、倉存數據的補錄。

## 4. 權限與審批流

### 4.1 級別權限設定 (RBAC)
系統在「資料輸入」模塊中定義角色與權限：
*   **銷售崗**：只能查看/編輯自己的客戶、報價單、合同；可查看產品目錄和可用庫存。
*   **庫管崗**：只能操作出入庫，查看庫存明細，無法查看銷售金額和財務報表。
*   **財務崗**：可管理請款單、預收款、AR/AP、總賬及所有財務報表。
*   **管理層**：擁有全量數據查看權限及高級審批權限。

### 4.2 審批流機制
*   **觸發條件**：請款單提交、超低價報價、大額採購等。
*   **流轉邏輯**：根據設定的「審批權限設置」，逐級流轉（如：部門經理 -> 財務主管 -> 總經理）。

## 5. 報表與分析

系統提供多維度的分析報告：
1.  **財務分析報告**：月度預算統計、收支趨勢、現金流預測。
2.  **會計報表**：利潤和損失表 (P&L)、企業資產負債表。
3.  **銷售分析報告**：銷售額統計、員工業績/佣金報表、產品暢銷榜。
4.  **客戶分析圖表**：客戶來源轉化率、客戶價值分析 (RFM)。
5.  **庫存成本查看**：庫存資金佔用、呆滯庫存預警。
