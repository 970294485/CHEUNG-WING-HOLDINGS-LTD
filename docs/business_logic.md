# 业务逻辑文档 (Business Logic)

> 本文档是「综合企业管理系统 (ERP/CRM)」的核心业务参考，涵盖财务、会计、销售、客户、产品、库存、文件管理及系统基础设置等全链路业务逻辑。
>
> **核心理念**：业财一体化、数据互通、权限严控、流程自动化。
>
> **读者**：产品、开发、运营及使用系统的企业员工。
>
> **相关文档**：`docs/tech.md`（技术规格）、`docs/development_phase.md`（开发阶段）

## 术语表

| 术语 | 说明 |
|------|------|
| **見積 (Quotation)** | 报价单，向客户提供的产品/服务价格明细 |
| **AR (Accounts Receivable)** | 应收账款，客户欠企业的货款 |
| **AP (Accounts Payable)** | 应付账款，企业欠供应商的货款 |
| **P&L (Profit & Loss)** | 利润和损失表（损益表），反映企业一定期间的经营成果 |
| **SKU** | 库存进出计量的基本单元 |
| **预收款 (Advance Payment)** | 在交付产品或服务前，提前向客户收取的款项 |

## 1. 系统总览

### 1.1 核心设计原则

| 原则 | 含义 |
|------|------|
| **业财一体化** | 销售、采购、库存的业务动作自动触发财务记账，减少人工干预 |
| **权限隔离** | 严格的级别权限设定，销售看客户，库管看库存，财务看账目 |
| **灵活可配** | 产品属性、会计类别、审批流等支持自定义，适应不同业务场景 |
| **数据资产化** | 完善的文件分类与网盘功能，沉淀企业知识与业务单据 |

### 1.2 系统模块定位

| 模块 | 说明 |
|------|------|
| **1.7 资料输入 (PT-Data Entry)** | 系统的基石，负责公司基本资料、权限、用户、产品主数据、期初数据的录入 |
| **1.6 客户管理 (Customer)** | 获客与维系，支持分组、跟进、邮件推广及分析 |
| **1.8 产品管理 (Product)** | 定义实物资产，支持多属性、多媒体、配件及价格体系 |
| **1.3 销售功能 (Sales)** | 交易闭环，从报价(見積) -> 合同 -> 预收发票 -> 佣金计算 |
| **1.9 库存管理 (Inventory)** | 实物管理，出入库操作、库存明细及成本核算 |
| **1.1 财务功能 (Financial)** | 资金流管理，请款、预收款核销、预算控制及审批流 |
| **1.2 会计功能 (Accounting)** | 账务核算，应收应付、总账、P&L、资产负债表 |
| **1.4 / 1.5 文件与导出 (Document/File)** | 单据 PDF/Excel 导出、个人网盘与公共文件数据库 |

## 2. 完整用户工作流

### 2.1 核心业务闭环（线索到现金 Lead to Cash）

```
1. 客户建档 → 录入客户信息、分组、分配跟进人 (1.6)
2. 销售报价 → 选择产品(1.8)，生成并导出报价单 PDF (1.3, 1.4)
3. 签订合同 → 报价转合同，确认交易细节 (1.3)
4. 预收与发票 → 生成预收发票，财务核销预收款 (1.1, 1.3)
5. 备货与采购 → 库存对接，库存不足则触发采购单 (1.3, 1.7, 1.9)
6. 发货出库 → 生成送货单(1.4)，扣减库存(1.9)
7. 财务结算 → 生成应收账款(AR)，确认收入，计算销售佣金 (1.2, 1.3)
8. 账务报表 → 数据自动流入总账，体现在 P&L 和资产负债表中 (1.2)
```

### 2.2 采购到付款闭环（Procure to Pay）

```
1. 采购需求 → 销售缺货或日常备货触发采购单 (1.7)
2. 采购入库 → 库管核对实物，操作入库，增加库存并核算成本 (1.9)
3. 财务请款 → 业务发起请款单，进入审批流 (1.1)
4. 审批与付款 → 财务/管理层审批通过，执行付款，核销应付账款(AP) (1.1, 1.2)
```

## 3. 核心模块业务逻辑详解

### 3.1 财务与会计模块 (Financial & Accounting)

**财务管理 (1.1)** 侧重于资金的计划与控制：
*   **请款与审批**：所有资金流出必须通过请款单发起，根据金额/类别触发预设的审批权限设置。
*   **预收款对接**：客户打款后，财务记录预收款；销售开具合同/发票时，系统支持将预收款与具体合同/发票进行核销对接。
*   **预算管理**：设定收入与支出预算，系统自动按月度统计实际发生额，生成预算执行对比报告。

**会计功能 (1.2)** 侧重于账务的准确记录：
*   **入账与类别**：基础数据中设定的会计入账项目与类别，用于规范所有收支记录。
*   **往来账 (AR/AP)**：销售出库自动增加应收；采购入库自动增加应付。
*   **核心报表**：系统根据总账记录，实时生成利润和损失表 (P&L) 及企业资产负债表。

### 3.2 销售与客户模块 (Sales & Customer)

*   **客户生命周期 (1.6)**：支持客户分组管理、跟进情况记录、来源追踪。集成 Email 发送推广讯息功能。提供客户分析图表。
*   **交易管理 (1.3)**：
    *   見積 (Quotation) -> 合同 (Contract) -> 预收发票 (Proforma Invoice) 的单据流转。
    *   **库存对接**：销售开单时实时查看库存，确认是否可售。
    *   **财务对接**：销售回款直接关联财务模块，并根据规则自动计算销售佣金。

### 3.3 产品与库存模块 (Product & Inventory)

*   **产品主数据 (1.8)**：
    *   基础字段：名称、SKU、Barcode、型号、规格、类型/子类型、重量、尺寸、包装尺寸、颜色。
    *   多媒体：支持图片、Video、附件上传。
    *   **高级特性**：支持产品配件设定；**产品属性支持自由增加并可以多选**（如自定义材质、适用场景等）。
    *   价格管理与导出（Excel/PDF）。
*   **库存管理 (1.9)**：
    *   管理出入库动作，提供详细库存台账。
    *   **成本查看**：结合采购入库价格，计算并展示库存成本（如移动加权平均）。

### 3.4 文件与系统基础 (Document, File & Data Entry)

*   **文件导出 (1.4)**：标准化输出发票、报价单、送货单、付款请求等（支持 Excel/PDF）。
*   **文件管理 1.0 (1.5)**：
    *   **个人网盘**：员工专属，支持上传、下载、档案分类和分享。
    *   **公共数据库**：企业知识库，支持把档案分享给内部员工。
*   **资料输入 (1.7)**：
    *   系统初始化入口：公司基本资料、单号规则、会计类目。
    *   权限控制：公司权限设定、级别权限设定、用户资料输入。
    *   历史/期初数据：报价单、合同、采购单、仓存数据的补录。

## 4. 权限与审批流

### 4.1 级别权限设定 (RBAC)
系统在「资料输入」模块中定义角色与权限：
*   **销售岗**：只能查看/编辑自己的客户、报价单、合同；可查看产品目录和可用库存。
*   **库管岗**：只能操作出入库，查看库存明细，无法查看销售金额和财务报表。
*   **财务岗**：可管理请款单、预收款、AR/AP、总账及所有财务报表。
*   **管理层**：拥有全量数据查看权限及高级审批权限。

### 4.2 审批流机制
*   **触发条件**：请款单提交、超低价报价、大额采购等。
*   **流转逻辑**：根据设定的「审批权限设置」，逐级流转（如：部门经理 -> 财务主管 -> 总经理）。

## 5. 报表与分析

系统提供多维度的分析报告：
1.  **财务分析报告**：月度预算统计、收支趋势、现金流预测。
2.  **会计报表**：利润和损失表 (P&L)、企业资产负债表。
3.  **销售分析报告**：销售额统计、员工业绩/佣金报表、产品畅销榜。
4.  **客户分析图表**：客户来源转化率、客户价值分析 (RFM)。
5.  **库存成本查看**：库存资金占用、呆滞库存预警。
